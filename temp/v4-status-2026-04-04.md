## Architecture (V4 — Correct Design)

- Hook does NOT override AMM. AMM runs real swaps.
- Hook manages LP bands (afterSwap rebalancing) + collects fees (beforeSwap).
- Fee: 1% on INPUT. Buys: quote asset to multisig. Sells: TRI burned.
- LP seeded one-sided (TRI only, $0 cost) — Clanker pattern.
- Ticks are NEGATIVE for USDC pool (-368400 base) due to 18/6 decimal mismatch.
- Permission bits: BEFORE_SWAP + BEFORE_SWAP_RETURNS_DELTA + AFTER_SWAP

## Current Blocker

**ChaosLP approval is failing.** USDC swap works fine. ChaosLP does not.

The error shows a multicall to the ChaosLP token's `allowance()` function reverting.
ChaosLP has `Permit2AllowanceIsFixedAtInfinity()` built into the token — calling
`approve(permit2, amount)` reverts. The `allowance()` call returns type(uint256).max.

Our code checks allowance before approving, which SHOULD skip the approve for ChaosLP.
But something in the flow is still hitting the revert. Possibilities:
1. The allowance check itself might be failing (multicall wrapper?)
2. wagmi/viem's readContract might be batching the call with multicall3
3. The Permit2.approve() step (step 2) might be failing, not the ERC20 approve

**USDC worked** — so the general flow is correct. ChaosLP is the specific problem.
