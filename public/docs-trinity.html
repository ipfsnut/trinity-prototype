Architecture

Trinity V8 replaces the discrete-band hooks (V6/V7) with a single continuous concentrated liquidity position per pool. No bands, no transitions, no afterSwap rebalancing. Each pool is managed by its own TrinityHookV8 instance.

  • One continuous CL position spanning a configured price range ($25k–$100M FDV)
  • External LP permanently blocked — only the hook can mint into the pool
  • Fees extracted before the swap via BeforeSwapDelta
  • No afterSwap logic — saves ~3000 gas per swap vs V6/V7
  • Fully parameterized: token, fee rate, range, and owner are constructor arguments

Arb Surface

3 pools across 3 quote assets create pairwise arbitrage surfaces. When any external asset moves, arb bots equalize pool prices — paying fees and burning TRINI in the process. Simulations showed 3–6% annual yield on seed capital in normal conditions, doubling during high-volatility periods.

V6/V7 → V8 Migration

V8 replaced the discrete-band architecture (V6/V7) which suffered from structural value leakage of ~50–65% on meaningful trades. All V6/V7 hooks have been drained. The TRINI ERC20 token and staking hub are unchanged.
